Detailed Description

The base class for an image editing tool. Every tool "has a" mask. Subclasses must define this mask by filling in the CreateMask factory method. This base class will then be able to apply the mask to a pixel buffer as the tool is dragged around interactively.

bool image_tools::Tool::applies_paint_when_stationary ( )
virtual

Most tools only apply "paint" when they are moved on the canvas, but some, like a spray can continue applying more paint over time even if they are held in the same place. Subclasses should override this function and return true if the tool allows paint to accumulate even when held stationary. The default implmentation returns false.

Reimplemented in image_tools::ToolSprayCan.

This function combines (i.e., blends) a paint_color with an underlying canvas_color given a mask_intensity. The default algorithm uses a standard blending function: paint_color * mask_intensity + canvas_color * (1.0 - mask_intensity) Some tools require a custom blend. Thus, this function provides a hook for subclasses to override the default blending function.

Reimplemented in image_tools::ToolChalk, and image_tools::ToolHighlighter.

ColorData image_tools::Tool::LookupPaintColor ( int  x,
int  y 
)
virtual

This function returns the current paint color used with the tool. In most cases, this is the same as the paint_color passed in to the StartStroke() function. However, some tools, like the blur tool, have "special paint" that depends on the underlying pixels in the canvas or some other factors. Thus, this function provides a hook for subclasses to override the default paint color to implement special paint colors.
